# 3. 지능형 스마트 업무 기능 요건

1. **사용자 편집이 가능한 파워포인트 보고서 추출**
   - 시스템의 스타트업 분석 데이터, 펀드 현황 등을 파워포인트 파일(PPTX)로 바로 다운로드합니다.
   - 이때 단순 화면 캡처 이미지가 아닌, 텍스트, 표, 네이티브 차트 객체 형태로 저장되어 다운로드한 사용자가 문서 프로그램을 통해 수치를 직접 편집하고 텍스트를 고칠 수 있어야 합니다.
   - **디자인 가이드라인 준수**: 생성되는 PPTX 내의 모든 텍스트, 표, 차트 범례 및 타이틀 등의 기본 폰트는 반드시 브랜드 표준인 **Pretendard (프리텐다드)** 단독 폰트로 고정하여 출력되도록 설계합니다.
   - **표준 PPTX 슬라이드 구성 템플릿**:
     * **1페이지: 타이틀 커버**
       * 브랜드 메인 컬러(`#515151`)배경 혹은 흰색 배경에 포인트 컬러(`#e22213`) 액센트 라인 적용.
       * 보고서 제목(예: "[스타트업명] 투자 검토 요약 보고서"), 출력 일자, 작성 심사역명 노출.
     * **2페이지: 기업 기본 개요 (표 형태)**
       * 스타트업 기본 정보(대표자명, 투자유치 단계, 설립일, 주요 비즈니스 모델 설명)를 격자 표(Table) 객체로 구성.
     * **3페이지: 시계열 성장 지표 (네이티브 차트)**
       * `startup_metrics` 테이블의 매출, 고용, 가치 변화 데이터를 묶어 꺾은선(Line) 또는 막대(Bar) 차트 객체로 동적 생성.
     * **4페이지: 주주 구성 및 후속 관리 현황**
       * 주주 비율 표 및 지분분율 텍스트, 주요 마일스톤 체크리스트와 분기 정기 보고서 제출 현황 리스트업.

2. **AI 대화형 파트너 (AI Agent)**
   - **데이터 통합 검토**: 대화형 인터페이스 내에서 로컬 파일(스타트업 IR 자료 PDF, 재무제표 엑셀 등)을 업로드할 수 있습니다.
   - **파일 및 데이터베이스 교차 분석 (RAG & DB Query)**:
     * **문서 파싱 및 임베딩 (Vector Store)**: 사용자가 업로드한 PDF/엑셀 문서는 텍스트 추출 가공 후 임베딩 모델(예: OpenAI `text-embedding-3-small`)을 거쳐 벡터 스토어에 임시 적재됩니다.
     * **하이브리드 조회**: AI 대화창 질문 발생 시, 업로드 문서 대상 시맨틱 검색(RAG)과 동시에 Supabase DB의 정형 데이터(해당 기업 정보, 매칭 풀, 진행 중인 프로그램 일정)에 대한 실시간 쿼리(또는 메타데이터 필터링)를 병합하여 컨텍스트를 구성합니다.
     * **보안 강화**: 업로드된 로컬 파일은 사용자·세션 단위 저장 경로로 격리하고 소유자와 Admin만 접근하도록 메타데이터 RLS를 적용합니다. 분석 완료 또는 보존 기간 만료 시 S3 Lifecycle Rule과 서버측 정리 작업으로 원본 및 임베딩을 함께 파기합니다.
     * **서버측 실행 원칙**: 파일 파싱, 임베딩 생성, DB 통합 조회와 모델 API 호출은 Edge Function 또는 별도 백엔드에서 수행합니다. 브라우저에는 OpenAI API Key, Supabase `service_role`, AWS 자격 증명을 노출하지 않습니다.
   - **지능형 업무 지원**: 교차 분석을 바탕으로 특정 스타트업에 최적화된 전문가 매칭 추천, 투자 유치 가능성 사전 진단, IR 요약 보고서 자동 생성 등의 다차원 피드백을 제공합니다.
     * **매칭 추천 프롬프트**: "이 스타트업의 비즈니스 분야(AI/바이오 등)와 현재 보육 프로그램의 연계 가능성 및 전문가 DB(`experts`) 내 멘토들의 전문분야 태그를 교차 분석하여 가장 만족도(`average_mentoring_rating`)가 높은 멘토 3명을 추천해줘."
