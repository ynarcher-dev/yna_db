# 🎨 공통 UI/UX 상태 및 레이아웃 가이드 (0_ui_ux.md)

본 문서는 **와이앤아처 데이터베이스 (PMS)** 시스템의 일관되고 프리미엄한 사용자 경험을 제공하기 위한 로딩/빈 화면/오류 상태 및 레이아웃 간격 표준을 정의합니다.

---

## 1. 데이터 조회 상태 및 UX 인터랙션

### 1.1 데이터 로딩 중 상태 (Loading State)
* **스켈레톤 UI (Skeleton)**: 테이블, 카드 그리드, 상세 프로필 등 레이아웃이 사전에 정의된 화면에서는 로딩 스피너 대신 스켈레톤 UI를 사용합니다.
  * **구현 규칙**: `Ant Design`의 `<Skeleton>` 컴포넌트 또는 Tailwind CSS의 `animate-pulse` 클래스를 사용해 회색 음영 블록이 깜빡이도록 렌더링합니다.
  * **적용 예시 (Tailwind)**:
    ```html
    <div class="animate-pulse flex space-x-4">
      <div class="rounded-full bg-yna-border h-10 w-10"></div>
      <div class="flex-1 space-y-6 py-1">
        <div class="h-2 bg-yna-border rounded"></div>
        <div class="space-y-3">
          <div class="grid grid-cols-3 gap-4">
            <div class="h-2 bg-yna-border rounded col-span-2"></div>
            <div class="h-2 bg-yna-border rounded col-span-1"></div>
          </div>
        </div>
      </div>
    </div>
    ```
* **전체 화면 스피너**: 페이지 전환 또는 데이터 제출(Submit)과 같이 컴포넌트 단위 로딩이 불가능할 경우에만 전역 로딩 오버레이를 사용합니다.

### 1.2 조회 결과 없음 상태 (Empty State)
* 데이터가 존재하지 않는 테이블, 칸반 보드, 자문 리스트 등의 화면에는 단순 빈 화면 대신 명확한 문구와 아이콘으로 비어 있음을 나타냅니다.
* **표준 구성**:
  1.  회색 무채색 아이콘 (예: `React Icons`의 `HiOutlineInbox` 또는 `Ant Design`의 `Empty` 컴포넌트)
  2.  직관적인 한국어 안내 메시지 (예: "등록된 스타트업 포트폴리오가 없습니다.")
  3.  **행동 유도 버튼 (CTA)**: 권한이 있는 경우 해당 데이터 추가 버튼을 하단에 제공합니다.
* **디자인**: 텍스트 색상은 `#515151` 대비 투명도를 낮추거나 서브 그레이(#8c8c8c)를 활용합니다.

### 1.3 비동기 오류 처리 및 사용자 피드백 (Error Handling)
* Supabase 데이터 쿼리 실패, 네트워크 단절 시 사용자가 인지할 수 있는 가시적인 알림을 노출합니다.
* **Try-Catch & Alert**:
  * 단순 조회 실패: 화면 상단에 3초간 노출되는 **Toast Alert** (Ant Design `message.error` 또는 React Hot Toast 등)로 안내합니다.
  * 치명적 예외 (권한 부족, 파일 삭제 오류 등): 사용자가 행동을 인지할 수 있도록 모달(Modal) 창을 띄워 상세 정보 및 재시도 옵션을 제공합니다.
  * **UI 문구 예시**: "데이터를 불러오는 중 오류가 발생했습니다. 잠시 후 다시 시도해 주세요." (시스템 로우 에러코드는 개발자 콘솔 `console.error`에만 기록)

### 1.4 얼럿 및 알림 시스템 유형과 가이드라인 (Alert & Notification UX Guidelines)
업무 처리 상태 및 신규 데이터 등록 상황을 알리기 위해, 개발팀 및 기획팀의 공용 용어인 **'얼럿(Alert)'**을 중심으로 아래와 같이 네 가지 표준 유형을 정의합니다.

*   **토스트 얼럿 (Toast Alert - 즉시 피드백)**
    *   **목적**: 사용자의 동작(생성, 수정, 복사 등)이 성공적으로 완료되었음을 화면 방해 없이 가볍게 피드백합니다.
    *   **UX 가이드라인**: 화면 우측 상단(또는 중앙 상단)에 3초 이내로 노출된 뒤 자동으로 사라지는 휘발성 얼럿입니다.
    *   **스타일**:
        *   `Success`: 브랜드 포인트 컬러 `#e22213` 또는 성공(Green) 계열 아이콘 활용
        *   `Info/Warning`: 무채색 메인 컬러 `#515151` 테마 활용
*   **다이얼로그/모달 얼럿 (Modal Alert - 명시적 경고 및 컨펌)**
    *   **목적**: 데이터 삭제, 중요 권한 회수 등 복구가 어렵거나 사용자 확인이 반드시 필요한 시점에 화면을 차단(Dim)하고 띄웁니다.
    *   **UX 가이드라인**: 확인/취소 등의 명시적인 버튼 클릭이 있기 전까지는 유지되며, 브라우저 기본 `alert()` 대신 커스텀 디자인된 모달 컴포넌트를 사용합니다.
*   **인라인 얼럿 (Inline Alert - 화면 내 정보/경고 배너)**
    *   **목적**: 특정 페이지 또는 입력 폼 본문 안에서 고정된 형태로 정보, 주의 사항, 필수 안내를 제공합니다.
    *   **UX 가이드라인**: Ant Design의 `<Alert>` 컴포넌트나 Tailwind CSS 배너 형태로 본문 영역 일부를 차지하며, 오류 상황 해결 전까지 화면에 고정 노출됩니다.
*   **배지 및 외부 얼럿 (Badge & External Alert - 미확인/시스템 알림)**
    *   **인앱 배지**: 사이드바 메뉴 및 헤더의 알림 종 아이콘 옆에 붉은색 숫자 배지로 표시됩니다.
    *   **외부 얼럿**: 임시 패스워드 발송 및 시스템 비상 이슈 발생 시 등록된 이메일 또는 슬랙(Slack)으로 얼럿 메시지를 전송합니다.

## 2. 간격, 타이포그래피 및 레이아웃 시스템

### 2.1 스페이싱 및 그리드 (Spacing Rules)
* 일관된 여백을 위해 **8px 배수 시스템 (Tailwind 단위 4배수)**을 준수합니다.

| 분류 | 간격값 | Tailwind 클래스 | 적용 예시 |
| :--- | :--- | :--- | :--- |
| **XS** | 4px | `p-1`, `m-1` | 배지 내부 간격, 컴포넌트 내 텍스트 간격 |
| **SM** | 8px | `p-2`, `m-2`, `gap-2` | 인풋 필드 간격, 테이블 셀 상하 여백 |
| **MD** | 16px | `p-4`, `m-4`, `gap-4` | 카드 컴포넌트 패딩, 리스트 아이템 간 여백 |
| **LG** | 24px | `p-6`, `m-6`, `gap-6` | 페이지 타이틀 하단 여백, 섹션 간 간격 |
| **XL** | 32px | `p-8`, `m-8` | 공통 레이아웃 사이드바-콘텐츠 외부 패딩 |

* **그리드 레이아웃**: 대시보드 및 카드 목록 뷰는 Tailwind CSS의 `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6`과 같이 반응형 그리드를 기본으로 탑재합니다.

### 2.2 모서리 둥글기 (Border Radius)
* 딱딱하고 권위적인 느낌을 줄이기 위해 부드러운 라운드 규격을 전역 컴포넌트에 할당합니다.
  * **버튼/인풋 필드**: `rounded-md` (6px 또는 8px) 적용
  * **카드 및 팝업 모달**: `rounded-lg` (12px) 또는 `rounded-xl` (16px) 적용
  * **아바타/프로필**: `rounded-full` (원형) 지정

### 2.3 타이포그래피 계층 (Hierarchy)
Pretendard 폰트의 가독성을 높이기 위한 글꼴 두께 및 크기 규칙입니다.

* **h1 (페이지 타이틀)**: `text-2xl font-bold tracking-tight text-yna-main` (24px)
* **h2 (섹션 타이틀)**: `text-lg font-semibold text-yna-main` (18px)
* **Body (본문)**: `text-sm text-[#515151]` (14px, 기본 텍스트)
* **Caption (서브 정보)**: `text-xs text-gray-500` (12px, 날짜 또는 작성자 정보)
